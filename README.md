# Dynamic Background (동적 배경 전환) — SillyTavern Extension

유저가 보낸 메시지든 AI(캐릭터)가 출력한 메시지든, 방금 새로 올라온 메시지 내용을 LLM에게 다시 보여주고,
등록해둔 배경 이미지 파일 목록 중 **가장 맥락에 어울리는 것을 AI가 직접 선택**하게 하는 방식입니다.
키워드를 일일이 정해둘 필요가 없습니다.

예: 유저나 캐릭터가 "방 안에 들어서자..." 같은 내용을 보내면, 목록에 있는 `room.png`가
(정확히 "방"이란 단어를 등록해두지 않아도) 문맥상 가장 어울린다고 판단되어 선택됩니다.

## 동작 방식

1. `CHARACTER_MESSAGE_RENDERED`(AI 메시지 렌더링)와 `USER_MESSAGE_RENDERED`(유저 메시지 렌더링) 이벤트를
   모두 감지해서, 누가 보냈든 방금 새로 올라온 메시지를 기준으로 판단합니다.
2. 해당 메시지 텍스트와, 설정에 등록된 배경 파일명 목록을 함께 SillyTavern의 `generateQuietPrompt()`로
   LLM에 전달합니다. (채팅 UI에는 보이지 않는 "조용한" 백그라운드 생성 요청입니다.)
3. Chat Completion API를 쓰는 경우 JSON Schema 구조화된 출력을 사용해서, 목록에 있는 파일명 중
   하나(또는 `NONE`)만 정확히 고르도록 강제합니다. 구조화된 출력을 지원하지 않는 API(Text Completion 등)라면
   일반 텍스트 응답에서 파일명을 추출하는 방식으로 자동 폴백합니다.
4. 선택된 파일로 배경을 부드럽게 크로스페이드 전환합니다. `NONE`이 선택되면 "기본 배경"으로 바꾸거나
   (설정된 경우) 현재 배경을 유지합니다.
5. 채팅을 전환하면(`CHAT_CHANGED`) 그 채팅의 마지막 AI 메시지를 기준으로 다시 계산합니다.

배경은 SillyTavern 자체 배경 시스템을 건드리지 않고, 확장이 추가한 전체화면 레이어(채팅 UI 뒤,
`z-index: -5`)에 표시됩니다.

## ⚠️ 비용/속도 관련 중요 참고

"메시지마다 자동 실행" 옵션을 켜두면, **유저 메시지든 AI 응답이든 메시지 1개당 배경 판단용 LLM 호출이 1번 추가로** 발생합니다.
API 과금이 있는 서비스를 쓰신다면 추가 비용이 들고, 응답 속도도 약간 느려질 수 있습니다.
부담스럽다면 자동 실행을 끄고, 필요할 때 설정 패널의 "마지막 메시지로 지금 다시 계산" 버튼으로 수동 실행하세요.

또한 유저가 메시지를 보내는 순간에도 배경 판단용 호출이 별도로 실행되는데, 이는 AI의 실제 응답 생성 요청과는
별개의 호출이라 서로 순서가 꼬이지는 않지만, 같은 타이밍에 API 호출이 두 번(배경 판단 + 실제 응답 생성) 겹쳐
나가는 셈이라 API 종류에 따라 응답이 살짝 느려질 수 있습니다.

## 설치 방법

1. 이 폴더(`st-dynamic-background`) 전체를 SillyTavern 서버의 아래 경로 중 하나에 복사하세요.
   - 사용자 전용: `data/<user-handle>/extensions/st-dynamic-background`
   - 전역(전체 사용자 공용): `public/scripts/extensions/third-party/st-dynamic-background`
2. SillyTavern을 새로고침하고, 확장(퍼즐 조각) 메뉴에서 "Dynamic Background"를 활성화하세요.
3. `index.js` 상단의 `EXTENSION_URL_PATH`가 실제 설치 경로와 일치하는지 확인하세요. (전역 설치 시 기본값 그대로 사용 가능)
4. `backgrounds/` 폴더 안에 실제 이미지 파일을 넣으세요.

## 사용 방법

1. 확장 설정 패널에서 "동적 배경 전환" 드로어를 엽니다.
2. "사용 가능한 배경 파일명 목록"에 `backgrounds/` 폴더에 넣은 파일명을 한 줄에 하나씩 입력합니다.
   (예: `room.png`, `forest.png`, `beach.png`, `night_street.png`)
3. 필요하면 "추가 지침"에 판단 기준을 자유롭게 적어줄 수 있습니다. (예: "애매하면 실내 배경 우선")
4. "AI 메시지마다 자동 실행"을 켜두면 매 응답마다 자동으로 배경이 바뀝니다. 끄면 수동 버튼으로만 동작합니다.
5. "마지막 메시지로 지금 다시 계산" 버튼으로 현재 상태를 바로 테스트해볼 수 있습니다.

기본값으로 `room.png / forest.png / beach.png / night_street.png / classroom.png` 예시 5개가 미리 들어있습니다.
실제 사용 전에 `backgrounds/` 폴더에 해당 파일이 있는지 확인하거나, 본인 목록으로 전부 바꾸세요.

## 한계 및 참고사항

- 파일명이 그 이미지의 의미를 나타내는 "라벨" 역할을 하므로, 파일명을 최대한 명확하게(예: `dark_forest_night.png`)
  지어두면 매칭 정확도가 올라갑니다.
- Text Completion 계열(로컬 모델, KoboldAI 등) API는 구조화된 출력을 지원하지 않을 수 있어 자동으로
  일반 텍스트 파싱 방식으로 전환됩니다. 이 경우 모델 성능에 따라 매칭 정확도가 다를 수 있습니다.
- 목록에 파일이 많아질수록 프롬프트가 길어져 판단 정확도가 떨어질 수 있으니, 20~30개 이내로 유지하는 것을 권장합니다.

## 파일 구성

```
st-dynamic-background/
├── manifest.json      # 확장 메타데이터
├── index.js           # 핵심 로직 (AI 기반 맥락 매칭, 배경 전환, 설정 UI)
├── style.css           # 배경 레이어 및 설정 패널 스타일
├── backgrounds/        # 배경 이미지를 직접 넣는 폴더
│   └── README.txt
└── README.md
```
